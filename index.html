一致性

一致性就是从分布式系统的任意副本读取的数据都相同。分布式系统最常用的两种是强一致性和最终一致性。

  • 强一致性要求写过的数据能被后续的所有读请求看到

面对对象

发表于 2016-10-10   |   分类于 软件架构   |  

面对对象编程3大特性：封装、继承、多态；五大原则：单一职责、开放封闭、里氏替换、依赖倒置、接口分离。最终目的是可维护、可扩展、可复用、灵活性好。

1.1 封装

讲到封装，我们先要想想什么是封装？为什么要封装？如何封装？

封装就是用一个类来描述“一个物”、“一个行为”，这个类起到隔离作用，对外提供访问接口，内部运行机制对用户透明。这样做有什么好处呢？这也是第二个问题的答案，这样做有以下优点：
1 隔离作用。可以保护内部私有数据不被外部改变，保护类的安全；同时用户也不需要关心事物内部的机制，降低复杂度。
2 解耦作用。实现业务分层处理，每层只关心该层的逻辑实现，而不需要知道下一层的内容。比如TCP/IP协议，TCP用于识别一台机器上的各个链路，只在数据包传到目的节点时才会用到，如果IP不对访问不了TCP层。如果没有封装，那所有MAC、TCP、IP的数据结构都会杂糅在一个结构体中。负责路由转发的节点就会同时处理所有的参数，从安全性、复杂度等方面看这显然是不可行的。

回调函数

发表于 2016-09-15   |   分类于 软件架构   |  

回调函数，顾名思义就是A1调用函数B，B还要返回来调用A2，其中A1和A2都是A的一部分。那B是怎么返回来调用A2的呢？就是通过A1传递给B的函数指针。形式上来说，回调函数都包含两个部分：函数指针赋值、函数指针执行。那回调函数的作用是什么呢？为什幺用回调函数呢？我的理解是增加灵活性，如果没有回调机制那STL中的sort等算法作用就不会这么强大了。

入职后解决的第一个疑难问题

发表于 2016-08-28   |   分类于 Linux开发   |  

前两天复现了一个部门疑难问题单，奖励了两百块钱，做为刚入职一个多月的新员工，感觉好爽~~

在这里不得不提一下我大华为的IT。华为的国际化远远领先国内所有公司，所以虽然华为在IT领域起步晚，但是我相信全球化带来的资源、市场优势能让华为在IT领域快速崛起。不进华为体验一下真感受不到全球化的优势。回归正题（^.^）

1 问题描述

当两个运行相同程序的进程的pid后12位相同时，产品就会出bug。

在分布式系统中区别进程的方式有多种。一个是建立一个集中管理的节点，但是这个又涉及到管理节点的可靠性问题，相当于一个小的分布式系统，这种方案太麻烦。另一个是根据ip地址和进程号直接可以定位唯一的进程。IP+PID做hash的key没有冲突。

我们的软件由于历史原因，字节限制只能使用28位做为key，其中16位用来存ip，还有12位存pid的后12位，这样就存在风险：后12位相同的进程产生的key会冲突，导致系统错误。

虚拟机内核crash分析

发表于 2016-08-28   |   分类于 Linux开发   |  

在云计算领域，虚拟机crash成为常态，串口输出是分析内核崩溃的手段之一，本文系统介绍了通过串口打印虚拟机信息的方法。

0 基础概念

  1. /dev/ttySn 或者/dev/ttysn是串口终端

  2. /dev/pty是伪终端，伪终端成对出现，对master的操作会反映到slave上。master在客户端，/dev/ptyp0；slave在服务器上，用/dev/ttyp0表示。对master的操作会反映到slave上。

  3. /dev/tty1-6 虚拟控制终端，常用的就是tty3和tty5。tty0是他们的别名。

  4. /dev/console 控制台终端，计算机显示器设备。

  5. /dev/tty 当前进程的控制终端，可以是以上任何一种终端
