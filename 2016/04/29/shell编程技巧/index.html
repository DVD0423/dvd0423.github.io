复杂的带变量的变量字符串可以用下面的方式确保它是字符串，这样在sed等里面就不会出错了

4 bash通配符

符号 意义
* 0到无穷多个任意字符
? 代表一定有任意一个字符
[abc] abc内的任意一个
[^abc] 非abc内的任意一个
[a-z] a到z

5 数据流重定向

  • /dev/null 垃圾桶，可以把标准输出和错误输出都放到这里
  • 1> 或者 > 覆盖标准输出到文件
  • 1>> 或者 >> 累加标准输出到文件
  • 2> 或者> 覆盖错误输出到文件
  • 2>> 或者>> 累加错误输出到文件
  • cat > ./myfile < diff.txt 把diff.txt输出到myfile文件
  • cat > ./myfile << "eof" 遇到eof则键盘输入结束
  • &> /dev/null 或者 > list 2>&1
  • 1> /dev/null 2> /dev/null
  • ls -l | tee myfile 输出结果存一份到myfile
  • ls -l | tee -a myfile 输出结果累加存一份到myfile

6 grep

参数或正则 意义
-v 反向查找
-n 显示行号
^word word在行首
word$ word在行尾
‘\.$’ 显示以”.”为结尾的行
. 一定有一个任意字符
* 重复0到无穷多个前一字符
.* 重复1到无穷多个前一字符
\{n,m\} 连续n到m个的前一个RE字符
\{n,\} 连续n个以上的前一个RE字符
\{n\} 连续n个前一个RE字符

7 sed

参数或正则 意义
-n 只有经过特殊处理的内容才会被显示
-i 直接修改读取的文件内容，而不是由屏幕输出;sed -i '$a # this is a test' myfile.txt在文件最后添加字符串# this is a test
ls | sed ‘2,5d’ 删除2-5行
ls | sed ‘2a fuck DW’ 在第二行加入字符串fuck DW
ls | sed ‘2,5c fuckDW’ 把2-5行替换成fuckDW
ls | sed -n ‘2,7p’ 显示2-7行，一般p和-n参数一起使用
sed ‘s/旧字符串/新字符串/g’ 字符串替换

其中’s/old/new/g’可以是’s#lod#new#g’中间的分隔符随意，默认是’/‘。但是如果old字符串或者new字符串里面有’/‘则会出错，这时要用其他字符(比如’#’)替代’/‘。

sed里面加变量的方式：
sed “s/$a/$b/g” …
eval sed ‘s/$a/$b/g’ …
sed ‘s/‘$a’/‘$b’/g’ …
sed s/$a/$b/g …

添加一行的方式：
‘2a …’ 是第二行后面添加 append
‘2i …’ 是第二行前面添加 insert

正则表达式使用方法和grep一样
